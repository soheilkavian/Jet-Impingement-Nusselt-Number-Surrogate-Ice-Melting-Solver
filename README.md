# Jet-Impingement Nusselt-Number Surrogate & Ice-Melting Solver

Public code and data accompanying the paper:

> **A physics-informed surrogate for free-surface jet-impingement heat transfer enabling fast simulation of ice melting in aircraft deicing**
> S. Kavian et al.

This package lets you reproduce, from a single self-contained folder, the three
building blocks of the paper:

1. a **data-driven neural network (NN)** surrogate for the radial Nusselt-number
   profile of a free-surface impinging jet,
2. a **correlation-anchored physics-informed network (PINN-A)** for the same task,
   which extrapolates far better in Reynolds number and yields a closed-form
   correlation, and
3. a **reduced-order axisymmetric ice-melting (Stefan) solver** that uses the PINN
   to supply the convective boundary condition and reproduces the turbulent
   melting field of **Figure 13**.

The surrogates are trained on a database of **168 COMSOL jet-impingement
simulations** spanning `Re = 5×10²–10⁵`, `Pr_jet = 3–500`, `Pr_wall = 3–10⁴` and
`H/d = 1–10`.

## Contents

| File | Description |
|------|-------------|
| `simulation_database.mat` | The 168-case COMSOL database (struct array `database`; per-case `Re, Pr_jet, Pr_wall, H_over_d, T_jet`, and radial profiles `r_over_d, Nu, h_Wm2K, T_wall`). |
| `prepare_dataset.m` | Shared loader: flattens the database, applies the log10 transform, and makes the case-grouped 80/20 train/test split (`rng(42)`). Used by both trainers. |
| `s01_train_NN.m` | **Code 1** — load the database and train the baseline NN. Saves `ml_Nu_baseline.mat`. |
| `s02_train_PINN.m` | **Code 2** — load the database and train the PINN-A. Saves `trained_models/pinnA_model.mat`. |
| `s03_predict_Nu_vs_rd.m` | **Code 3** — load a trained model (asks NN or PINN at start) and plot `Nu` vs `r/d` for a laminar and a turbulent example. |
| `s04_melt_turbulent_Fig13.m` | **Code 4** — load the PINN and animate the turbulent liquid-fraction melting field over time (reproduces the model column of Fig. 13). |
| `predict_pinnA.m` | Helper: evaluate the trained PINN-A on raw features `[Re, Pr_jet, Pr_wall, H/d, r/d]`. |
| `ml_Nu_baseline.mat` | A **pre-trained NN** (so Code 3 runs out of the box). Regenerated by Code 1. |
| `trained_models/pinnA_model.mat` | A **pre-trained PINN-A** (so Codes 3 & 4 run out of the box). Regenerated by Code 2. |

## Requirements

* **MATLAB R2021b or newer** (developed/tested on R2022b).
* **Statistics and Machine Learning Toolbox** (provides `fitrnet`).
  *Code 1 falls back to `feedforwardnet` from the Deep Learning Toolbox if
  `fitrnet` is unavailable.*

No other dependencies. COMSOL is **not** required — the simulation database is
shipped with the package.

## Quick start

Open MATLAB, `cd` into this folder, and run the scripts in order:

```matlab
% (optional) re-train the surrogates from the database
s01_train_NN        % -> ml_Nu_baseline.mat
s02_train_PINN      % -> trained_models/pinnA_model.mat

% predict the radial Nusselt profile (laminar + turbulent examples)
s03_predict_Nu_vs_rd        % choose [1] NN or [2] PINN when prompted

% animate the turbulent ice-melting field (Fig. 13)
s04_melt_turbulent_Fig13
```

Pre-trained models are included, so you can run **Code 3** and **Code 4**
immediately without first running Codes 1–2.

## The PINN-A model

The physics-informed surrogate predicts the local Nusselt number as

```
log10(Nu) = Phi(X) * A_coef  +  NN_residual( z(X) )
```

with the closed-form (Sieder–Tate-style, stagnation-regularized) basis

```
Phi(X) = [ 1, log10(Re), log10(Pr_jet), log10(Pr_jet/Pr_wall),
           log10(H/d), log10(r/d + 1) ]
```

`A_coef` is fitted by ordinary least squares; the `[32 32]` tanh residual network
learns the small remaining deviation. To evaluate it on your own inputs:

```matlab
M  = load('trained_models/pinnA_model.mat');
X  = [Re, Pr_jet, Pr_wall, H_over_d, r_over_d];   % N x 5
Nu = predict_pinnA(X, M);                          % N x 1
```

## The ice-melting solver (Code 4)

`s04_melt_turbulent_Fig13.m` solves an axisymmetric enthalpy (Stefan) problem on
a 2-D `(r,z)` finite-volume grid. Each time step it (a) locates the melt front,
(b) calls the PINN for the convective coefficient `h(r,t) = Nu·k/d` at the moving
interface (the local stand-off `H/d` grows as the cavity deepens), (c) updates the
enthalpy by explicit time stepping, and (d) applies a soft "wash-out" relaxation of
fully-liquid cells toward the jet state (open, constantly refreshed jet). The
turbulent operating point is `v_jet = 14 m/s`, `T_jet = 80 °C`; edit the header to
try other conditions (e.g. `v_jet = 8`, `T_jet = 40` for the laminar companion).

Runtime is ~30–60 s on a 160×160 grid to `t = 5 s`. Reduce `nr`, `nz` in the
script for a faster preview.
